# Debugging Lab

This lab gives learners practical failure scenarios instead of abstract troubleshooting advice.

## Scenarios

- wrong Jenkins agent label
- agent offline
- missing credential ID
- Docker permission denied
- incorrect Git repository URL
- Java version mismatch
- Jenkins disk full
- test report missing
- pipeline timeout
- stale workspace
- port already in use
- incorrect ownership of `JENKINS_HOME`

## How to Use This Lab

For each scenario:

1. Read the problem statement.
2. Inspect the listed logs and commands.
3. Identify the root cause.
4. Apply the corrective action.
5. Add a prevention note.
